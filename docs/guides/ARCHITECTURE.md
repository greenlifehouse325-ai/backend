## Scalability Considerations

### Horizontal Scaling

- **Stateless design**: Semua state disimpan di Supabase
- **JWT tokens**: Validasi tanpa database lookup (kecuali fresh user check)
- **No session storage**: Server tidak menyimpan session

### Performance Optimizations

- **Connection pooling**: Supabase handle connection pooling
- **Indexed queries**: Refresh tokens indexed by user_id dan expires_at
- **Lazy token validation**: Skip database check jika token valid

### Future Considerations

- Redis untuk caching dan rate limiting
- Message queue untuk async tasks
- Microservices split jika diperlukan
